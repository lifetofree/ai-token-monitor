# AI Token Monitor 📊

A real-time token monitor dashboard analyzing cost, usage, and caching optimization across multiple LLM brands including **Gemini**, **Claude**, **Minimax**, and **GLM**.

This is a local, lightweight tool designed for personal development environment tracking to ensure operations stay within budgets using rolling limits. v1 traffic is **synthetic** (in-app Simulation) — see [Real Mode removal](./docs/adr/0005-remove-real-rtk-mode.md) for the rationale.

## 💡 Core Concepts

### 1. Token Metrics & Caching Efficiency
The dashboard computes costs and savings using three primary metrics from the LLM requests:
* **Input Tokens**: The prompt size sent to the model.
* **Output Tokens**: The response size generated by the model.
* **Saved Tokens**: Tokens served from the local caching proxy (modelled synthetically in v1).
* **Savings %**: Evaluated as `Saved / (Input + Saved)`.

### 2. Live Rolling Windows & Sliding Treadmill
Rather than using absolute calendar-day resets, the spend safety limits operate on a **sliding/rolling window**:
* **5-Hour Window**: Tracks the cumulative cost of requests made in the last 5 hours.
* **Weekly Window**: Tracks the cumulative cost of requests made in the last 7 days.
* **Treadmill Expiry**: As time progresses, individual requests "fall out" of the window. A countdown timer displays exactly when the oldest active expense will expire.

### 3. In-App Simulation (v1 Data Source)
The dashboard is **Simulation-only** in v1. There is no external database reader and no live proxy integration. The simulator generates synthetic traffic every 8-20s with randomised input/output token counts and cache hit rates. The "Send Custom Request" modal lets you fire a one-off Request with chosen parameters.

> The previous Real RTK Monitor Mode (which read from `~/Library/Application Support/rtk/history.db`) was removed — see [`docs/adr/0005-remove-real-rtk-mode.md`](./docs/adr/0005-remove-real-rtk-mode.md). Re-introducing it would be a future module with its own ADR.

## ⚙️ Features & Architecture

* **Multi-Brand Support**: Customized pricing rates and budgets for **Gemini**, **Claude**, **Minimax**, and **GLM**.
* **Synthetic Traffic Simulation**: Generates organic background requests on a randomized 8-20s schedule for local layout and logic testing. A "Send Custom Request" modal can fire one-off Requests.
* **Spend Safeguards**: Tracks costs within a rolling **5-Hour** and **Weekly** window, prompting warning/danger status bars as thresholds are reached.
* **Zero Dependencies**: Pure HTML5, CSS custom variables (supporting light/dark themes), and vanilla client/server JavaScript.
* **Security Baselines**: Strict CORS rules (restricted to localhost), path traversal protection, env-var whitelist for the `.env` writer, and a per-key endpoint that returns masked values.
