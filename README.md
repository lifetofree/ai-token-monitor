# AI Token Monitor 📊

A real-time token monitor dashboard analyzing cost, usage, and caching optimization across multiple LLM brands including **Gemini**, **Antigravity**, **Claude**, **Minimax**, and **GLM**.

This is a local, lightweight tool designed for personal development environment tracking to ensure operations stay within budgets using rolling limits.

---

## 🚀 Getting Started

### Prerequisites
* **Node.js** (v14+)
* **SQLite3** command-line interface installed (if using Real RTK Monitor mode)
* **RTK (Rust Token Killer)** proxy binary set up (for real-mode tracking)

### Running Locally
Initialize the local development server:
```bash
npm run dev
# or
node server.js
```
The application will automatically attempt to open [http://localhost:3000](http://localhost:3000) in your default browser.

## 💡 Core Concepts

### 1. Token Metrics & Caching Efficiency
The dashboard computes costs and savings using three primary metrics from the LLM requests:
* **Input Tokens**: The prompt size sent to the model.
* **Output Tokens**: The response size generated by the model.
* **Saved Tokens**: Tokens served from the local caching proxy (e.g., via the RTK wrapper). Caching saves up to 60-90% on developer costs by reusing prompt prefixes.
* **Savings %**: Evaluated as `Saved / (Input + Saved)`.

### 2. Live Rolling Windows & Sliding Treadmill
Rather than using absolute calendar-day resets, the spend safety limits operate on a **sliding/rolling window**:
* **5-Hour Window**: Tracks the cumulative cost of requests made in the last 5 hours.
* **Weekly Window**: Tracks the cumulative cost of requests made in the last 7 days.
* **Treadmill Expiry**: As time progresses, individual requests "fall out" of the window. A countdown timer displays exactly when the oldest active expense will expire.

### 3. RTK (Rust Token Killer) Database Integration
When running in **Real RTK Monitor Mode**, the Node.js server polls the local SQLite database at `~/Library/Application Support/rtk/history.db`:
* Shell operations that invoke LLMs are recorded by the RTK tool.
* The local Node.js server queries this SQLite database and transmits command history, token metrics, and execution speeds safely to the frontend stream.

## ⚙️ Features & Architecture

* **Multi-Brand Support**: Customized pricing rates and budgets for Gemini, Antigravity, Claude, Minimax, and GLM.
* **Dual-Mode Operation**:
  * **Simulation Mode**: Generates mock organic background API requests for local layout and logic testing.
  * **Real RTK Mode**: Connects to the local RTK SQLite history database to parse actual LLM commands run via the command-line hook.
* **Spend Safeguards**: Tracks costs within a rolling **5-Hour** and **Weekly** window, prompting warning/danger status bars as thresholds are reached.
* **Zero Dependencies**: Pure HTML5, CSS custom variables (supporting light/dark themes), and vanilla client/server JavaScript.
* **Security Baselines**: Avoids shell injection via direct argument execution, implements strict CORS rules (restricted to localhost), and blocks path traversal attacks.
