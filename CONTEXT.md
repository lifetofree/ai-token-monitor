# AI Token Monitor

The ubiquitous language of the AI Token Monitor: the terms a domain expert uses to describe what the dashboard shows and what the user does. Scope is the dashboard's data model, not the implementation.

> v1 is **Simulation-only**. The Real RTK Monitor Mode and the `source` attribute on Request were removed in `docs/adr/0005-remove-real-rtk-mode.md`. The remaining terms describe a single-mode system whose traffic comes from the in-app simulator.

## Language

**5-Hour Window**:
A fixed sliding time window of five hours used to compute accumulated Cost and trigger Rolling Spend Limit breaches.
_Avoid_: 5h window, rolling 5h

**Brand**:
An LLM provider tracked by the dashboard, identified by a stable string key. v1 supports four: `gemini`, `claude`, `minimax`, `glm`.
_Avoid_: Provider, Model, Vendor
Aliases: "Provider" is acceptable in user-facing copy but never in data fields.

**Brand Metadata**:
The per-Brand configuration: display name, Pricing, 5-Hour Spend Limit, and Weekly Spend Limit.

**Cache Hit Rate**:
The fraction of a Request's input that was served from cache, expressed as `savedTokens / (inputTokens + savedTokens) * 100`. A percentage.
_Avoid_: cache rate, hit ratio

**Cached Token**:
A token that was served from cache and not billed. Represented as `Request.savedTokens`, disjoint from `Request.inputTokens` once `docs/adr/0003-cache-model-disjoint-input-and-saved.md` is fully applied (see ADR-0003 status — not yet applied in code).
_Avoid_: saved token, free token

**Cost**:
The dollars attributed to a Request, computed from Pricing as `(inputTokens * inputRate + outputTokens * outputRate) / 1_000_000`. The dashboard never knows what was actually charged; this is a working estimate.
_Avoid_: Actual Cost, Billed Cost

**Input Token**:
A token sent to the LLM as part of the prompt. Billed at the Brand's input rate.
_Avoid_: prompt token

**Output Token**:
A token generated by the LLM as part of the completion. Billed at the Brand's output rate.
_Avoid_: completion token, response token

**Pricing**:
A Brand's per-million-token input and output rates, configured by the user. Drives Cost. The dashboard never reconciles with actual invoices.
_Avoid_: rates, list price

**Request**:
One LLM API call as recorded by the dashboard. Has a Brand, timestamps, `inputTokens`, `outputTokens`, `savedTokens`, and a computed Cost and Savings. In v1, every Request is produced by the in-app simulator; the `source` attribute is reserved for a future Real Mode re-introduction (see `docs/adr/0005-remove-real-rtk-mode.md`).
_Avoid_: call, invocation, command, transaction

**Rolling Spend Limit**:
A cap on accumulated Cost over a fixed sliding time window. A Brand has exactly two: 5-Hour and Weekly.

**Savings**:
The dollars avoided on a Request because of cache hits, computed as `savedTokens * inputRate / 1_000_000`. A cross-cutting metric on Request, not a property of any Brand.
_Avoid_: saved cost, cache savings

**Simulation**:
The single Monitor Mode in v1. Generates synthetic traffic on a randomized 8-20s schedule. Useful for development, demo, and the absence of real RTK data.
_Avoid_: sim mode, mock mode, simulator (a verb, not the noun)

**Weekly Window**:
A fixed sliding time window of seven days used to compute accumulated Cost and trigger Rolling Spend Limit breaches.
_Avoid_: 7d window, rolling weekly

## Relationships

- A **Request** belongs to exactly one **Brand** (via `brand`).
- A **Brand** has exactly one **Brand Metadata** (one Pricing, one 5-Hour Spend Limit, one Weekly Spend Limit).
- A **Brand** has two **Rolling Spend Limits** (5-Hour Window, Weekly Window).
- A **Cached Token** belongs to exactly one **Request** and is disjoint from that Request's **Input Tokens** (once ADR-0003 is fully applied).

## Example dialogue

> **Dev:** "What data source drives the dashboard?"
> **Domain expert:** "In v1, only the in-app Simulation. The Real RTK Mode and the mode switcher were removed — see ADR-0005."

> **Dev:** "If a Request hits the cache, how do I compute the Cost?"
> **Domain expert:** "Each Request records `inputTokens` (billed) and `savedTokens` (cached) as disjoint fields. Cost = `(inputTokens * inputRate + outputTokens * outputRate) / 1M`. Cache Hit Rate = `savedTokens / (inputTokens + savedTokens)`. Caveat: the disjoint cost formula is accepted but not yet applied in code — see ADR-0003 status."

> **Dev:** "What does the 'Resets at HH:MM' countdown mean?"
> **Domain expert:** "It's not a real reset. With sustained traffic, the Rolling Spend Limit slides continuously — the oldest in-window Request falls out at that moment, and the next-oldest becomes the new boundary."

> **Dev:** "Why is there no 'antigravity' Brand?"
> **Domain expert:** "It was a synthetic entry representing personal-optimized traffic. Dropped in ADR-0001. The four real Brands are gemini, claude, minimax, and glm."

> **Dev:** "Is there a 'mode' attribute on a Request?"
> **Domain expert:** "Not in practice. The store is the `state.requests` array; every Request is produced by the simulator. The schema reserves a `source` field for a future Real Mode re-introduction, but the current renderer doesn't filter on it."

## Flagged ambiguities

- **"Antigravity"** appeared in two roles: project author and a Brand key. Resolved: the Brand `antigravity` is dropped from the data model. The author name is orthogonal to the dashboard's Brand list.
- **"Actual Cost"** was a column header that implied the dashboard knew what was actually charged. Resolved: it never did. The label was redundant; the column is now `Cost`, computed from configured Pricing.
- **`meta.limit`** is a per-Brand field that is never read by the renderer. **Status: still present in `DEFAULT_BRAND_METADATA`; deletion tracked in `docs/REVIEWS.md` R3.**
- **`windowLabel`** is a per-Brand cosmetic string that suggests the window length is configurable. **Status: still present in `DEFAULT_BRAND_METADATA` and still read by `renderBrandCards()`; deletion tracked in `docs/REVIEWS.md` R3.** Windows are fixed at 5h and 1w.
- **The cache model** is internally inconsistent: `billedInput = inputTokens - savedTokens` (subset model) coexists with a Cache Hit Rate formula that treats the two as disjoint. **Status: accepted in `docs/adr/0003-cache-model-disjoint-input-and-saved.md`, not yet applied in code.** The cost formula still uses the subset model; the rate formula ranges 0-100% as if disjoint.
- **"Provider" vs "Brand"** are used interchangeably in UI copy. Resolved: the canonical term is Brand. "Provider" survives in user-facing strings (e.g. the simulation modal) but never in data fields.
- **Real Mode vs Simulation Mode** were two Monitor Modes. Resolved by `docs/adr/0005-remove-real-rtk-mode.md`: Real Mode is removed; v1 is Simulation-only. The `source` attribute on Request is reserved for a future re-introduction.
- **Two stores (`state.requests` vs `state.realCommands`)** were collapsed into one. Resolved by `docs/adr/0002-unify-request-stores-by-source.md` and reinforced by `0005`.
